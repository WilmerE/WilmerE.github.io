Sistema Publiprim

Sistema de gestión de pedidos para una imprenta en Guatemala, desarrollado en PHP con arquitectura MVC

Descripción General

Sistema de gestión de pedidos para una imprenta en Guatemala, desarrollado en PHP con arquitectura MVC. Incluye gestión completa de pedidos, clientes, usuarios, notificaciones en tiempo real y sistema de roles.

Arquitectura

PHP MVC MySQL/MariaDB jQuery DataTables Select2 WebSocket Font Awesome
  • Backend: PHP con arquitectura MVC
  • Base de datos: MySQL/MariaDB
  • Frontend: jQuery, DataTables, Select2, Font Awesome
  • Tiempo real: WebSocket para notificaciones instantáneas
  • Config: Variables de entorno seguras

Estructura de Datos Principal

Tablas principales:

  • Clientes - Información de clientes
  • Pedidos - Gestión de órdenes con seguimiento
  • Ventas - Ítems y detalles de cada pedido
  • Cobros - Pagos y abonos
  • Usuarios - Control de acceso con roles
  • Notificaciones - Centro de notificaciones
  • Tareas - Notas compartidas del equipo
  • Archivos - Carpeta compartida de documentos

Sistema de Tracking

Los pedidos tienen 7 estados de tracking (0-6):

  • 0. Anulado
  • 1. Entregado
  • 2. Avisar
  • 3. Impreso
  • 4. Confirmado
  • 5. Diseñado
  • 6. Pendiente

Sistema de Roles

  • Superuser: Acceso total
  • Staff: Acceso a tracking completo
  • Vendor: Acceso a Anulado, Entregado, Avisar, Diseñado, Pendiente
  • Designer: Solo Diseñado y Pendiente
  • Printer: Impreso y Confirmado

Sistema de Notificaciones en Tiempo Real

  • Servidor WebSocket - Sistema de mensajería en tiempo real
  • Cliente WebSocket - Envío de mensajes al servidor
  • Acciones notificables:
    • Actualización automática de tablas de pedidos
    • Notificaciones instantáneas de cambios
    • Sincronización de notas compartidas

Funcionalidades Principales

Gestión de Pedidos

  • Crear/editar pedidos con ítems y abonos
  • Cambio de estado (tracking)
  • Generación de PDF
  • Sistema de carpetas compartidas automático

Centro de Notificaciones

  • Registro de todas las acciones del sistema
  • Badge con contador de notificaciones no vistas
  • Auditoría de cambios

Notas Compartidas (ToDo)

  • Lista de tareas pendientes compartida
  • Estado done/undone

Gestión de Usuarios

  • CRUD de usuarios
  • Asignación de roles y permisos

Cotizaciones

  • Sistema de cotizaciones

Deudores

  • Control de cuentas por cobrar

Helpers y Utilidades

  • Formateo automático de IDs de pedidos
  • Formateo de fechas y horarios
  • Sistema de gestión de notificaciones
  • Cliente para comunicación en tiempo real

Frontend

  • Diseño responsive con menú toggle
  • DataTables para tablas dinámicas
  • Select2 para selectores mejorados
  • Indicadores visuales por estado de tracking
  • Actualización en tiempo real vía WebSocket

Autenticación

  • Sistema de login seguro
  • Encriptación de contraseñas
  • Sistema de permisos por rol
  • Gestión de sesiones

Topología del Sistema en Producción

El sistema está desplegado en una red local con 4 estaciones de trabajo, cada una con acceso personalizado según su rol:

Estación de Ventas

  • Atención al cliente
  • Creación de pedidos
  • Gestión de cobros
  • Seguimiento completo de órdenes

Estaciones de Diseño (x2)

  • Vista de pedidos pendientes
  • Cambio a estado "Diseñado"
  • Acceso a archivos compartidos
  • Notificaciones de nuevos trabajos

Estación de Impresión

  • Vista de pedidos confirmados
  • Control de trabajos impresos
  • Actualización de estados
  • Gestión de producción

Todas las estaciones se sincronizan en tiempo real mediante WebSocket, permitiendo que cualquier cambio realizado en una computadora se refleje instantáneamente en las demás, optimizando el flujo de trabajo del equipo.
